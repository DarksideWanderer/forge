# Forge 包管理器 TOML 配置规范

本文档定义了 Forge 包管理器的配置文件（通常为 `forge.toml`）的结构和参数说明。

## 1. 基础配置 `[package]` (必须)

定义包的基本元数据。

- **name**: (必须) 字符串。项目的名称，只能包含字母、数字、下划线和连字符。
- **version**: (必须) 字符串。语义化版本号（SemVer），例如 `"0.1.0"`。
- **authors**: (可选) 字符串数组。作者列表。

## 2. 编译器配置 `[compiler]`

定义项目全局的编译器行为。

- **name**: (可选) 字符串。指定编译器名称，如 `"clang"` 或 `"gcc"`。默认值为 `"clang"`。
- **cpp_standard**: (可选) 字符串。C++ 标准版本，如 `"c++20"`, `"c++23"`。默认值为 `"c++20"`。
- **stdlib**: (可选) 字符串。指定标准库，如 `"libc++"` 或 `"libstdc++"`。 默认值为 `"libc++"`。
- **flags**: (可选) 字符串数组。全局编译选项，如 `["-Wall", "-Wextra"]`。默认值为 `["-Wall", "-Wextra"]`。

## 3. 构建配置 `[build]`

定义构建系统的全局参数。

- **type**: (可选) 字符串。构建类型，通常为 `"release"` 或 `"debug"`。默认值为 `"release"`。
- **target**: (可选) 字符串。目标架构/系统，如 `"x86_64-linux"`, `"arm64-macos"`, `"x64-windows"`。(暂定, 可能实现自动探测)
- **system**: (可选) 字符串。默认构建驱动系统，如 `"cmake"`, `"meson"`, `"make"`, `"autotools"`。默认值为 `"cmake"`。

## 4. 特性开关 `[features]`

定义可选的功能模块。

- **modules**: (可选) 布尔值。全局模块化开关。
- **tests**: (可选) 布尔值。是否开启测试支持。(这个的实现方式是, 直接用一个test.cpp 直接探测所有的测试框架, 只要能编译通过就说明支持测试了, 这样就不需要用户手动指定测试框架了, 暂定是否实现这个功能)

## 5. 依赖管理 `[dependencies.<name>]`

定义项目所需的外部依赖。`<name>` 为依赖包的名称。

- **version**: (必须) 字符串。依赖的版本号。
- **build_system**: (可选) 字符串。该依赖使用的构建系统（可覆盖全局 `[build].system`）。
- **build_type**: (可选) 字符串。该依赖的构建类型（可覆盖全局 `[build].type`）。
- **flags**: (可选) 字符串数组。专门用于该依赖的编译参数。
- **modules**: (可选) 布尔值。该依赖是否开启模块化支持。// 暂定

---

## 示例配置文件

```toml
[package]
name = "my-project"
version = "1.0.0"

[compiler]
name = "clang"
cpp_standard = "c++20"

[dependencies.fmt]
version = "10.2.1"
build_system = "cmake"
```
